一.服务治理.
   在传统得rpc调用中,随着服务逐渐增多,服务与服务之间得依赖关系更加复杂,也更加难以管理,所以需要使用服务治理.
   也就是管理服务之间得依赖关系,可以实现服务注册与发现,调用,负载均衡与容错等.
   
   而在早期得SpringCloud中就是使用Euraka来实现服务治理.
   
二.Euraka作用简介.
   Euraka采用得CS得架构设计,Euraka Server 作为服务注册功能的服务端,是服务注册中心,而系统中的其他微服务,通过Euraka的客户端连接到Euraka Server 并维持心跳连接,这样系统维护人员就能通过
   Euraka来监控系统中各个微服务是否正常运行.
   
   在服务启动后,其他微服务(服务提供者)会将服务器信息,比如通信地址以别名的方式注册到注册中心上,服务调用者再通过别名找到实际地址,实现rpc本地调用.在任何rpc远程调用中都会有一个注册中心.
   
   
三.Euraka两组件
   1.Euraka Server
    各个微服务通过配置启动后,	会在Euraka Server中进行注册,这样Euraka Server会在服务注册表中存储所有可用服务节点的信息,通过界面可以直观看到.
   2.Euraka Client
     是一个Java客户端,用于简化和服务端的交互,所有的消费者和服务提供者都相当于一个Euraka Client端
