一.存储过程.
1.存储过程是什么.
   就是sql语句的一个集合,可以认为是sql执行程序的一个方法,一个函数,可以定义方法参数,返回值.

2.优点
   经过一次编译后不需要再次编译,可直接执行.
   方便执行批量的重复使用的sql操作,并具有原子性.
   增强了sql语句的灵活性,流控制语句编写(ifelse)，有很强的灵活性，可以完成复杂的判断和较复杂的运算

3.和事务的区别

   不同点:
          存储位置,事务一般是保存在项目中,而存储过程一般作为数据库中的一个函数,保存在数据库中
          事务有严格的一致性和原子性,具有极大的安全性,存储过程则没有,在进行一些复杂的操作中,通常在存储过程中使用事务.



二.视图.
1.视图是什么?
   就是从一个表或多个表中通过一些查询虚拟出来的表.
   举例:
         创建视图,从a表,b表,c表中抽取一个字段作为视图.
          create view other as select a.name as username, b.name as goodsname from user as a, goods as b, ug as c where a.id=c.userid and c.goodsid=b.id;
         查询视图.
          select * from other;

2.作用.
   1.其实感觉就像程序中的一个对象一样，我们会有一些执行语句去生成一个对象，定义了一个方法，那当我们需要这个对象的时候，直接调用这个方法即可。
      这个方法可以在很多地方进行复用。当需要修改时，只需要修改这个方法即可，调用者是不需要修改的。
   2.安全性设置。
      视图可以有自己的权限设置。并且和表的权限设置不冲突。
